## Friction, deviations, skips
- **FR-133 plugin packaging:** `@blockly/zoom-to-fit@13.1.0` + `@blockly/workspace-minimap@13.1.0` ship UMD dists; bundling them into editor-ui's ESM dist synthesized a broken `blockly/core` default-import interop that failed editor-element's IIFE build AND its Vitest suite. Fix: treat both plugins as external runtime deps exactly like `blockly` itself (external in editor-ui/react/element ESM builds; real deps of element/react so the IIFE self-bundles). **Watch-out:** the plugins' peer range is `^13.1.0` while the repo pins `blockly@13.0.0` — pnpm warns (non-fatal); consider bumping blockly to 13.1.0 at the next AD-021 pin review.
- **Real bug found only in the browser pass:** the minimap mirrors the primary workspace by replaying change events, but the mount's programmatic mutations (`loadDocument`/`clear`) run with Blockly events disabled — so the mirror stayed empty and every later replay threw `The associated block is undefined` (console error on each collapse/move/delete). Fixed with `SyncablePositionedMinimap.syncFrom()` (subclass reaches the protected `minimapWorkspace`; full-state reload after each programmatic mutation), red-first test in `navigation.test.ts`. jsdom suites had only asserted the minimap's DOM presence — a lesson for what "functional" must mean in future navigation tests.
- **Blockly implicit default:** `collapse` is already on for any `categoryToolbox` + non-read-only inject; the explicit option is contract-hardening, not a behavior change (documented in mount.ts + test).
- **Density harness split:** the plan placed the corpus sweep in `packages/editor-ui/test/`, but `encode()` needs a real engine (AD-008/AD-032) and editor-ui must not depend on the adapter (dep cycle) — the full-corpus sweep lives in `test/engine-node-adapter/test/ui/density-corpus.test.ts` (per-file jsdom override), the committed baseline stays at `packages/editor-ui/test/density-baseline.json`.
- **28px was already met pre-compaction** (26px) — the compact renderer's value is the corpus-wide density gain (median bbox height −14.2%, area −15.9%) + headroom (20px), not rescuing the bound.
- A stale `editor-ui/dist` initially made the density numbers look like a no-op (the adapter harness resolves the built dist, and direct `npx vitest --root` bypasses turbo's `test → ^build`); caught by the implementer, rebuilt, honest before/after captured. The same stale-dist trap hit the browser pass once (fix invisible until `pnpm --filter @transon/editor-ui build`).
- Synthetic-event automation could not activate Blockly's context-menu items or tooltips through the preview tool; a real (trusted-event) Playwright click worked. ArrowDown highlighting worked, Enter did not — menu activation needs trusted events.
- Cosmetic: `Can't delete non-existent block` stderr in the density harness's clear-reload cycling (Blockly-internal; assertions unaffected) — likely reduced by the minimap resync fix. The zoom-to-fit control renders with `aria-label` missing (`button: "undefined"` in the a11y tree) — plugin-internal DOM; recorded as a follow-up nit, not patched.
